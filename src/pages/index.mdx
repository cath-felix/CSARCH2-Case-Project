  # 🎮 Texture Filtering in 3D: From Pixels to Reality

  ## Introduction

  When you play a game or watch a 3D movie, textures (the images wrapped around 3D objects) need to be displayed correctly at various distances and angles. This process is called **texture filtering** — a critical function of modern GPUs.

  ---

  ## Why Texture Filtering Matters

  Without texture filtering, 3D graphics would look:
  - **Pixelated** when zoomed in (blocky squares)
  - **Aliased** (jagged edges) when viewed at angles
  - **Distorted** when objects are far away

  ---

  ## The 4 Main Texture Filtering Methods

  | Method | Quality | Performance | How It Works |
  |--------|---------|-------------|--------------|
  | **Nearest Neighbor** | Low | Fastest | Uses the closest pixel (no blending) |
  | **Bilinear** | Medium | Good | Averages 2x2 surrounding pixels |
  | **Trilinear** | High | Heavy | Bilinear + MIPMAP blending |
  | **Anisotropic** | Best | Most demanding | Accounts for viewing angle |

  ---

  ## How Texture Filtering Works in the GPU Pipeline

  ### Step 1: Texture Coordinates (UV Mapping)
  Each 3D object has UV coordinates that map its surface to a 2D texture image.

  ### Step 2: Pixel Mapping
  The GPU calculates which texture pixels (texels) correspond to each screen pixel.

  ### Step 3: Filtering Applied
  - **Minification** (texture too small → stretched): Uses MIPMAPs
  - **Magnification** (texture too big → zoomed in): Blends nearby texels

  ### Step 4: Final Pixel Color
  The filtered result is displayed on screen.

  ## MIPMAPs: The Performance Hack

  **MIPMAPs** are pre-scaled copies of a texture (like a pyramid). When an object is far away, the GPU uses the smaller version instead of shrinking the full texture.
