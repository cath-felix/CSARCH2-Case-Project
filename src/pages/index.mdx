# 🎮 How GPU Filter Textures: Nearest Neighbor vs Bilinear Filtering

## Introduction

When a GPU renders a 3D scene, it needs to apply textures to objects. But what happens when a texture is viewed at different distances or angles? The GPU uses **texture filtering** to determine pixel colors — and two of the most fundamental methods are **Nearest Neighbor** and **Bilinear Filtering**.

---

## What is Texture Filtering?

Texture filtering is the process of **sampling a texture image** and calculating the correct color for each pixel on screen. It's a critical GPU function that affects:

- **Visual Quality** — sharp or blurry?
- **Performance** — fast or slow?
- **Memory Usage** — how much data is needed?

---

## The Problem: Pixels vs Texels

| Term | Definition |
|------|------------|
| **Pixel** | A single dot on your screen |
| **Texel** | A single dot in a texture image |
| **The Problem** | 1 pixel doesn't always match 1 texel (especially when zooming) |

When a texture is displayed larger or smaller than its original size, the GPU must **interpolate** (calculate) the missing pixels.

---

## Nearest Neighbor Interpolation

**How it works:** The GPU simply picks the closest texel to the sample point. No averaging, no blending.

**Formula:** P(x,y) = P(round(x), round(y))

### ✅ Advantages:
- **Fastest** filtering method
- Simple to implement
- Preserves sharp edges

### ❌ Disadvantages:
- **Blocky**, pixelated appearance when zoomed in
- Noticeable aliasing (jagged edges)
- Poor quality for realistic images

### 🎮 Where it's used:
- Pixel art games (like Minecraft)
- Retro-style games
- Older hardware

---

## Bilinear Filtering

**How it works:** The GPU averages the 4 nearest texels (a 2x2 grid) and blends them together.

**Formula:** P(x,y) = (TL + TR + BL + BR) / 4

### ✅ Advantages:
- **Smooth** appearance
- No obvious pixelation
- Better visual quality

### ❌ Disadvantages:
- **Slower** than Nearest Neighbor
- Can appear slightly blurry
- Uses more GPU resources

### 🎮 Where it's used:
- Most modern games
- 3D applications
- Textures viewed at normal distances

## Comparison Table

| Feature | Nearest Neighbor | Bilinear |
|---------|------------------|----------|
| **Quality** | Low (blocky) | Medium (smooth) |
| **Performance** | Fastest | Good |
| **GPU Usage** | Very Low | Low |
| **Memory Usage** | None | None |
| **Formula** | Closest texel | (TL + TR + BL + BR) / 4 |

---

## When to Use Each Method

| Scenario | Recommended Method | Why |
|----------|-------------------|-----|
| **Pixel art games** | Nearest Neighbor | Preserves blocky aesthetic |
| **Retro style games** | Nearest Neighbor | Authentic retro look |
| **Modern 3D games** | Bilinear | Good balance of quality/speed |
| **VR/AR applications** | Anisotropic | Best quality for angles |
| **Mobile games** | Bilinear | Performance-focused |

---

## How This Works in the GPU Pipeline

1. **Texture Loading** — GPU loads texture into memory
2. **UV Mapping** — Texture coordinates are calculated
3. **Sample Point** — GPU finds where to sample on texture
4. **Filtering Applied** — Nearest or Bilinear (or other)
5. **Pixel Output** — Final color is displayed on screen

---

## Fun Facts

- **Nearest Neighbor** was the default in early 3D games (like PlayStation 1)
- **Bilinear** became popular with the PlayStation 2 era
- Modern GPUs can process **100+ billion texels per second**
- **Anisotropic** filtering (the next step up) was introduced in 1999 with NVIDIA GeForce 256
